ALGORITHM FEATURES IMPLEMENTED
================================================================================

LAYER 0: CIRCUIT BREAKER (Emergency Stop)
  ✅ 3 states: CLOSED (normal), HALF_OPEN (degraded), OPEN (blocked)
  ✅ Dynamic activation on recent SR < 30%
  ✅ Auto-recovery after cb_recovery_rounds transactions
  ✅ Configurable threshold, eval window, recovery period
  ✅ Penalty for half-open state (0.15 default)

LAYER 1: CONTEXT SEGMENTATION
  ✅ Independent bandit per context
  ✅ Context key: payment_mode | issuing_bank | amount_bucket
  ✅ Lazy initialization (no pre-allocation)
  ✅ Automatic context detection from transaction context
  ✅ Amount bucketing: 0-500, 500-5k, 5k-50k, 50k+

LAYER 2A: SLIDING WINDOW UCB
  ✅ Sliding window of last W transactions (default: 200)
  ✅ Exploration bonus: √(2 × ln(N_total) / n_window)
  ✅ Fast reaction to crashes (seconds-level)
  ✅ Configurable window size (10-10,000)

LAYER 2B: THOMPSON SAMPLING
  ✅ Beta(α, β) Bayesian posterior
  ✅ Stochastic sampling for exploration
  ✅ Natural confidence quantification
  ✅ Time decay (exponential forgetting)
  ✅ Handles delayed feedback elegantly
  ✅ Configurable priors and decay rate

LAYER 2C: DISCOUNTED UCB
  ✅ Exponential decay of old observations
  ✅ Effective memory: 1/(1-γ) transactions
  ✅ Smooth adaptation to gradual drift
  ✅ Configurable decay factor (0.01-1.0)

LAYER 3: ENSEMBLE COMBINATION
  ✅ Weighted blend of 3 algorithms
  ✅ Production-optimized weights (Dream11 data)
  ✅ SW_component = 0.60 × SW-UCB + 0.40 × TS
  ✅ Final = 0.70 × SW_component + 0.30 × D-UCB
  ✅ Half-open penalty (-0.15)

STATE MANAGEMENT
  ✅ Per-context, per-gateway state tracking
  ✅ SW-UCB: deque of window outcomes
  ✅ TS: alpha/beta counters
  ✅ D-UCB: discounted sum/count

================================================================================
HYPERPARAMETER CONFIGURATION
================================================================================

11 CONFIGURABLE HYPERPARAMETERS

Sliding Window UCB (Layer 2A):
  window_size (default: 200, range: 10-10,000)
    - Memory horizon in transactions
    - Lower = faster adaptation, noisier
    - Higher = smoother, slower reaction

Thompson Sampling (Layer 2B):
  ts_alpha_prior (default: 1.0, range: 0.01-100)
    - Beta distribution prior success parameter
  ts_beta_prior (default: 1.0, range: 0.01-100)
    - Beta distribution prior failure parameter
  ts_decay (default: 0.995, range: 0.99-0.999)
    - Exponential decay for old observations
    - Lower = forgets faster, higher = trusts history

Discounted UCB (Layer 2C):
  discount_factor (default: 0.70, range: 0.01-1.0)
    - Exponential decay rate (γ)
    - Lower = 3.3 effective txns, higher = slower forgetting

Ensemble Weights (Layer 3):
  ucb_weight (default: 0.60, range: 0.0-1.0)
    - 0.60 × SW-UCB + 0.40 × TS_sample
  sw_weight (default: 0.70, range: 0.0-1.0)
    - 0.70 × SW_component + 0.30 × D-UCB

Circuit Breaker (Layer 0):
  cb_threshold (default: 0.30, range: 0.10-0.70)
    - SR below which circuit opens
  cb_eval_window (default: 20, range: 5-100)
    - How many transactions to evaluate
  cb_recovery_rounds (default: 200, range: 10-2,000)
    - How long circuit stays open
  degraded_penalty (default: 0.15, range: 0.0-1.0)
    - Score reduction for half-open gateways

  Can be customized for specific data distributions
